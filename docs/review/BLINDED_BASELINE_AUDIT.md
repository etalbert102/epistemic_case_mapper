# Blinded Baseline Audit

Status: `human-review-needed`

Purpose: audit whether the existing decision-space erosion findings still survive when the comparator is the local Gemma4 blinded baseline rather than the original illustrative baseline written in the same Codex context as the curated maps.

Blinded baseline files:

- `examples/lhc_black_holes/blinded_flat_synthesis_baseline_gemma4.md`
- `examples/eggs/blinded_flat_synthesis_baseline_gemma4.md`

Protocol:

- Comparator baselines were generated by `scripts/run_blinded_baselines.py`.
- Model: `gemma4:e4b` through local Ollama.
- Baseline prompt used only selected raw source text line spans.
- The prompt did not expose curated maps, erosion audits, best-region indexes, judge walkthroughs, or source-packet crux/loss guidance.
- This audit is still agent-authored and must not be treated as human review.

Decision labels:

- `survives`: the erosion loss remains fair against the blinded baseline.
- `survives_with_revision`: the loss remains directionally fair, but the description should be narrowed because the blinded baseline preserved some of the lost item.
- `weakened`: the loss is much less compelling against the blinded baseline and should not be counted without human review.
- `rejected`: the blinded baseline preserves the allegedly lost item well enough that the loss should not be counted for this comparator.

Eggs summary:

- Strongly survives: `eggs_loss_005`, `eggs_loss_006`, `eggs_loss_007`.
- Survives only with narrower wording: `eggs_loss_003`.
- Weakened: `eggs_loss_004`.
- Should not be counted against the blinded baseline: `eggs_loss_001`, `eggs_loss_002`.

Interpretation: the eggs case becomes a more demanding test under a blinded local-model comparator. Gemma4 preserved the biggest study-design and endpoint distinctions better than the original illustrative flat synthesis. The remaining evidence for decision-space erosion shifts toward guideline-process provenance, evidence-grade limitations, subgroup specificity, replacement specificity, and similar-wording scope distinctions.

## Overall Finding

The blinded baselines strengthen the prototype scientifically because they expose which erosion claims are robust and which were artifacts of the original illustrative comparator.

Measured against the blinded Gemma4 baselines:

- LHC: 5 of 6 losses remain countable if two are narrowed; 1 should be rejected for this comparator.
- Eggs: 4 of 7 losses remain countable if one is narrowed; 1 is weakened; 2 should be rejected for this comparator.

This improves the submission posture, but it also forces a more honest claim: structured maps are not always better because every ordinary synthesis loses every major distinction. Rather, structured maps provide a reusable audit surface that reveals which distinctions a given synthesis preserved, flattened, or omitted.

## Recommended Updates Before External Submission

- Add a blinded-comparator column to each erosion audit instead of replacing the original illustrative audit.
- Revise `lhc_loss_004`, `eggs_loss_001`, and `eggs_loss_002` so they are no longer counted as surviving losses against the Gemma4 blinded baseline.
- Ask a human reviewer to audit the `survives_with_revision` and `weakened` decisions before making quantitative claims.
- Consider generating blinded baselines from at least one additional local model family to check whether these survival rates are model-specific.
